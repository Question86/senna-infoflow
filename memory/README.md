# Senna Memory

This directory is the public, repository-backed memory surface for `senna-infoflow`.

It is not a secret store and not a surveillance layer. It is a structured place for durable, action-relevant knowledge about:

- GitHub findings and monitor results
- AXI0M / User Yps project developments
- infrastructure decisions
- expansion vectors
- recurring risks, opportunities, and follow-up tasks
- links into the dedicated long-term keeper repository

## Boundary

Allowed:

- public repositories and configured public sources
- user-provided notes and decisions
- summaries of findings generated by `senna-infoflow`
- implementation notes and issue/commit references

Not allowed:

- tokens, credentials, cookies, private keys, recovery codes
- private personal data about third parties
- unverified accusations as facts
- scraped private material or access-bypassed data
- raw sensitive payloads that should remain outside the repo

## Keeper bridge

The long-term memory repository is:

- Repo: `Question86/keeper-clean`
- Branch observed: `master`
- Role: concentrated long-term memory for GitHub findings, developments, projects, infra vectors, and Senna/AXI0M build history

`senna-infoflow` should keep a small, readable bridge here and refer to `keeper-clean` for deeper durable state.
