This directory is the Senna Infoflow subcategory for durable finding observation.

It uses the keeper-clean structure style, but it lives in the Senna Infoflow repo because these findings are part of the information system, not the Keeper Clean template.

---

## Purpose

Keeper Zenner stores concise, durable notes about findings discovered by Senna Infoflow and follow-up research.

It is for things that are too useful to lose, but not yet ready to become alerts, products, or decisions.

## Store here

- interesting GitHub/tooling discoveries
- early trend indicators
- public signal events worth watching
- first-pass research conclusions
- opportunities and risks for Yps / AXI0M
- watch triggers and next actions

## Do not store here

- secrets, tokens, credentials, cookies, private keys
- private third-party personal data
- doxxing, stalking, or access-bypass material
- raw noise dumps without distillation
- betting advice or financial recommendations
- unverified accusations written as facts

## Writing test

A finding enters here only if it answers yes to:

> Will Yps likely benefit from remembering this later?

Each entry should have:

1. What it is.
2. Why it matters.
3. What is known.
4. What is probable.
5. Chance for Yps / AXI0M.
6. Risk.
7. Watch triggers.
8. Next action.
9. What not to do.

## Core files

- `INDEX.md` - quick map and current status
- `findings/CHEATSHEET.md` - compact watchlist of notable findings
- `findings/agent_ops_watchlist.md` - thematic watchlist for Agent Ops signals
